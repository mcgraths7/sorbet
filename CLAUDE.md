# CLAUDE.md — Sorbet

Sorbet is a fun, themeable, accessibility-first design system: a Sass + TS core
with React bindings, in an npm-workspaces monorepo. The prefix everywhere is
`sb-` (`.sb-*`, `--sb-*`, `data-sb*`, `@layer sb.*`, the `sb:theme` event, the
`sb-theme` localStorage key).

## Packages

- `packages/design-system` (`@sorbet/design-system`) — framework-agnostic core.
  - `src/tokens/` — TS source of truth: `color.ts` (OKLCH→sRGB + WCAG contrast
    math), `ramps.ts`, `scales.ts` (incl. the `misc` dimension map), `semantics.ts`
    (contrast-driven semantic color builder), `presets.ts`
    (sorbet|ocean|forest|noir|midnight), `charts.ts` (8-slot CVD-validated chart
    palettes — slot ORDER is CVD-load-bearing and must stay identical across
    modes; re-validate if touched), `rules.ts` (the WCAG contract; `Rule.mode`
    gives per-mode minimums: chart slots light ≥3, dark ≥2.25).
  - `src/styles/` — Sass, atomic layers loaded into CSS cascade layers via
    `meta.load-css` in `index.scss`. Token access goes ONLY through the
    validated functions in `abstracts/_tokens.scss` — `clr()`, `space()`,
    `fs()`, `radius()`, …, and `token()` for one-off dimensions from the `misc`
    map (`border-width`, `control-height-md`, `sidebar-width`, `avatar-size-*`,
    `tile-min`, …). Every accessor, including `token()`, validates its name
    against generated lists at compile time; a raw `var(--sb-…)` or a literal
    dimension in a partial is a smell. Component knobs (un-prefixed custom
    props like `--gap`, `--min`, `--size`, `--aside`, `--drawer-size`) are
    per-instance API: they fall back to tokens and are reset with
    `--knob: initial;` in layout primitives so parent values never leak in.
    `abstracts/_generated.scss` is GENERATED by `tools/build-tokens.ts` —
    never hand-edit it; edit `src/tokens/scales.ts` and rebuild.
  - `src/behaviors/` — vanilla TS; `init()` wires `[data-sb=…]`. Zero deps.
  - `tools/` — `build-tokens.ts` (imports `../src/tokens/index.ts` via Node 24
    type-stripping; writes `dist/themes/`, `manifest.json`, `_generated.scss`;
    FAILS THE BUILD on contrast violations) and `check-contrast.ts`.
  - Exports: `.` & `./tokens`, `./behaviors`, `./css`, `./themes/*`, `./scss/*`.
- `packages/component-library` (`@sorbet/component-library`) — React 19, one
  subpath per atomic layer: `/core /layout /atoms /molecules /organisms
  /templates /charts`; the root barrel re-exports everything;
  `sideEffects: false`. Internal cross-layer imports are RELATIVE
  (`"../core/index.ts"`). `ThemeProvider`/`useTheme` live in `/core` (framework
  service); `ToastProvider` lives in `/molecules` (a provider ships with its
  UI) — both placements are deliberate. Shared combobox machinery:
  `src/molecules/combobox-core.ts` (`useComboboxCore`). Chart series colors
  resolve through `src/charts/shell.tsx` (`seriesColor`, `mutedSeriesColor`) —
  the only place chart `var(--sb-chart-*)` strings live. Styling belongs in
  the Sass partials; inline `style` in components is reserved for setting
  knob custom props from props (`--min`, `--aside`, `--ratio`, …).
- `packages/cli` (`@sorbet/cli`) — bin `sorbet` (create/theme/component/
  presets/contrast). `create` copies design-system sources + `scaffold/`
  templates into a standalone project; scaffold files are inert in place
  (relative imports) and reuse `emit.ts`/`scales.ts`, so token changes
  propagate automatically.
- `apps/playground` — Vite React kitchen sink (port 5183, strict). `demo/` —
  vanilla kitchen sink served from repo root (expects :4181). Both consume the
  published custom-property API directly — raw `var(--sb-*)` is correct THERE
  (no Sass compile at runtime); the accessor-function rule applies to the
  library's Sass, not to consumers.
- `.claude/launch.json` (gitignored): `playground` (npm run dev -w playground)
  and `sorbet-demo` (python3 -m http.server 4181 -d .).

## Commands

- Build all: `npm run build` (design-system → component-library → cli; includes
  the WCAG gate — an inaccessible palette fails the build).
- `npm run check:contrast` — full per-preset/mode report.
- Node via nvm: prefix PATH with `$HOME/.nvm/versions/node/v24.17.0/bin`.
- gh CLI at `/opt/homebrew/bin/gh` (authed as mcgraths7).

## Reuse before building (scan first)

Before adding a component, behavior, or style block, grep the tree for the same
shape. If the same logic or CSS already lives in ~2+ places, extract it into a
shared unit FIRST, then build on it — don't paste a fourth copy. The Popover
refactor is the template: `positionPopover` + `usePopover` were lifted into
`core/` and now back the menu, the pickers, and the generic Popover atom.

- Shared TS lives in `core/`: `usePopover`/`positionPopover` (anchored-panel
  lifecycle + placement), `composeRefs`, `cx`, `PolymorphicProps` (the `as`
  prop), `ThemeProvider`/`useTheme`. Combobox machinery is
  `molecules/combobox-core.ts` (`useComboboxCore`); chart series color resolves
  through `charts/shell.tsx`. Put a new cross-layer hook/util in `core/`.
- Shared Sass lives in `abstracts/_mixins.scss`: `focus-ring`, `elevate`,
  `pressable`, `truncate`, `control-reset`, `visually-hidden`, `respond`. A
  repeated animation/surface block (e.g. the popover flyout) belongs here as a
  mixin, not copy-pasted per partial.
- Layout primitives (`Stack`/`Cluster`/`Grid`/`Split`/…) are for COMPOSING
  layouts (organisms, templates, consumers). Atom/molecule internals stay BEM
  `<div class="sb-*__part">` styled by a Sass partial — styling lives in Sass,
  so don't swap a `.sb-card__body` for a `<Stack>`. Do prefer a real semantic
  element over `role`-on-`div` when one exists (`<ul>`/`<li>` for a legend).

## Pre-1.0: reshape freely, don't force sharing

Before 1.0 nothing here is frozen — components and these rules are all mutable
when a better pattern emerges, so prefer reshaping over duplicating. When two
close components can't share as-is, first try WIDENING the shared unit: give a
mixin a parameter, a component a prop, or a helper an option so both use it
neatly. But don't force it — if making two things share takes real contortions,
they're distinct enough to keep their own code.

## Conventions

- Layout owns ALL spacing; atoms/molecules have zero margins.
- Platform first: native `<dialog>`, Popover API, `@starting-style`,
  `:user-invalid`, `field-sizing`; React components are thin typed wrappers
  over the `sb-` classes.
- Fixed-position popovers (combobox/menu) dismiss on outside scroll
  (capture-phase listener; panel-internal scroll exempt).

## Browser-pane testing gotchas (this dev environment)

- Screenshots are BLANK whenever `scrollY > 0` → verify with DOM geometry or a
  `translateY` shift on `.sb-app-shell`.
- rAF / ResizeObserver / scroll-event delivery starves between tool calls →
  take a screenshot to pump a frame, then read state.
- Each `javascript_tool` call implicitly clicks the page (light-dismisses
  popovers); React commits for untrusted events are deferred → do the action
  in one call, read results in the next.
